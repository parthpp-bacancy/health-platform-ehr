# Frontend Design Skill

## Purpose
Build user-facing UI for the virtual care platform using Next.js App Router, Tailwind, and shadcn/ui.

## Required steps
1. Read `/doc/PRD.md`, `/doc/TASKS.md`, and `/doc/DECISIONS.md`.
2. Identify the route group and feature boundary.
3. Default to React Server Components; add `'use client'` only when needed.
4. Use Zod-backed forms with React Hook Form.
5. Add loading, empty, and error states.
6. Keep components focused and accessible.
7. Write a handoff record to `/doc/PROGRESS.md` before returning.

## Project-specific conventions
- Patient, provider, and org-admin surfaces should be visually distinct but use the same design system.
- Prefer co-located `_components/` under feature routes for private UI.
- Shared feature components belong in `components/[feature]/`.
- Never hand-edit `components/ui/` files generated by shadcn.

## When to load references/
- Load `references/component-patterns.md` when creating new reusable components.
- Load `references/tailwind-conventions.md` when spacing, density, or token consistency matters.
